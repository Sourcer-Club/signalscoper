# SignalScoper

**What they know vs what they do.**

SignalScoper is a free, open-source role intelligence tool for recruiters. Instead of matching job description keywords, it surfaces the persona signals, responsibility-level vocabulary, and technical markers that candidates actually use to describe their own work — the language that lives in LinkedIn headlines, profile summaries, and resumes, not in hiring manager briefs.

Search any of 228 roles across Technology, BFSI, Healthcare & Pharma, and Manufacturing & Engineering. Each role returns persona keywords, technical signals, synonyms, and a ready-to-use Boolean search string. A built-in SkillLens layer surfaces 110 secondary skills with recruiter-facing signal guides and Boolean strings.

Part of the [Sourcer.club](https://sourcer.club) suite — free tools for serious sourcers.

---

## Using the tool

SignalScoper is a single HTML file with no dependencies, no backend, and no build step.

**Online:** [signal.sourcer.club](https://signal.sourcer.club)

**Offline / self-hosted:**
1. Download `index.html`
2. Open it in any modern browser
3. No server required

---

## Features

- 228 roles across Technology, BFSI, Healthcare & Pharma, Manufacturing & Engineering
- Persona keywords — responsibility-level vocabulary candidates use, not JD language
- Technical signals — tools, platforms, and frameworks with SkillLens integration
- Two-tab keyword system — persona signals and technical signals separately
- Filter by specialisation — language, domain, or sub-discipline where relevant
- Boolean string generator — persona and technical strings, copy with one click
- SkillLens — 110 secondary skills with signal guides and Boolean strings
- India-first — role titles, regulatory bodies, certification norms, and tool prevalence reflect the Indian market
- Light and dark mode
- No login, no tracking, no data sent anywhere

## Contributing

Found a role missing? Keyword that doesn't belong? Signal that should be there?

Use the **Share a suggestion** button inside the tool. Every submission is reviewed.

To contribute directly — fork the repo, edit the role data in `index.html`, and open a pull request with a note on what you changed and why.
